# Инструкция по сборке проекта «Будильник PRO» в .exe файл

Сборка программы в исполняемый `.exe` файл позволяет запускать приложение на любом компьютере с Windows, даже если на нём не установлены Python, Pygame или другие используемые библиотеки.

---

### Этап 1. Защита Git-репозитория от мусора компилятора

В процессе сборки утилита `PyInstaller` создает сотни временных тяжелых файлов и кэшей. Чтобы они не захламляли вкладку Git в PyCharm, их нужно скрыть.

1. Откройте файл `.gitignore` в корне вашего проекта.
2. Допишите в самый конец следующие строки:
   ```text
   build/
   dist/
   *.spec
   ```
3. Зафиксируйте изменения в Git через интерфейс PyCharm (`Ctrl + K` / `Cmd + K`), написав сообщение: `chore: ignore pyinstaller build artifacts`.

---

### Этап 3. Установка компилятора и запуск сборки

1. Откройте встроенный **Терминал (Terminal)** внизу экрана PyCharm.
2. Проверьте, что вы находитесь внутри виртуального окружения (в начале строки терминала должно быть написано `(.venv)`).
3. Установите утилиту сборщика:
   ```bash
   pip install pyinstaller
   ```
4. Запустите сборку одной точной командой:
   ```bash
   pyinstaller --noconsole --onedir --icon="app_icon.ico" --name="AlarmClockPRO" main.py
   ```

#### Разбор флагов команды:
* `--noconsole` — полностью отключает фоновое черное окно консоли Windows при старте. Будильник запустится как стандартное чистое графическое приложение.
* `--onedir` — собирает приложение в **одну папку** (директорию). Это самый надежный способ компиляции программ со звуком (Pygame) и системным треем (Pystray), защищающий от потери динамических библиотек операционной системой.
* `--icon="app_icon.ico"` — берет нашу сгенерированную иконку и «вшивает» её прямо в тело файла `.exe` в качестве официального логотипа.
* `--name="AlarmClockPRO"` — задает красивое имя для итоговой папки и исполняемого файла.

### Этап 4. Обход «ловушки» относительных путей...

После завершения компиляции в проекте появятся папки `build/` и `dist/`. Готовый файл будет лежать по адресу: `dist/AlarmClockPRO/AlarmClockPRO.exe`.

Если запустить его прямо сейчас, программа откроется, но **звук работать не будет**, а база данных будильников не сможет загрузиться. Это может произойти потому, что программа ищет ресурсы *рядом с запущенным файлом*, а папка сборки пока пуста.

#### Чтобы приложение ожило:
1. Перейдите в корень вашего Python-проекта.
2. Скопируйте файл **`alarm.mp3`** (дефолтный трек) и папку **`music/`** (куда копируются кастомные мелодии).
3. Перейдите в папку **`dist/AlarmClockPRO/`**.
4. Вставьте скопированный файл и папку прямо туда — в одну директорию с файлом `AlarmClockPRO.exe`.
